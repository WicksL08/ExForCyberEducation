// key vocabulary
OSINT

Gathering information from publicly available sources without unauthorized access.

Ex: searching social media, public records, maps
Digital Footprint

The trail of data left behind when using the internet — public posts, accounts, locations.

Ex: tagged photos, check-ins, usernames
Verification

Confirming that found information is accurate by cross-referencing multiple sources.

Ex: checking three sources before concluding
Metadata

Hidden data embedded in files that can reveal creation date, location, and device info.

Ex: GPS coordinates in photo files

// key vocabulary
Encoding

Converting data into a different format for transmission — not a security measure, easily reversed.

Ex: Base64 encoding
Hashing

A one-way transformation that produces a fixed-length output. Cannot be reversed.

Ex: SHA-256 hash
Encryption

Two-way protection using keys — data can be encrypted and later decrypted by authorized parties.

Ex: AES, RSA encryption
Caesar Cipher

A substitution cipher that shifts each letter by a fixed number of positions in the alphabet.

Ex: A→D, B→E with shift of 3

// key vocabulary
Salting

Adding a random string to a password before hashing to prevent rainbow table attacks.

Ex: password + random_salt → hash
Rainbow Table

A precomputed table of hash values used to reverse-lookup and crack unsalted passwords.

Ex: hash lookup attack on MD5
Brute Force

Trying every possible combination of characters until the correct password is found.

Ex: trying aaa, aab, aac…
Credential Reuse

Using the same password across multiple sites — makes users vulnerable if one site is breached.

Ex: same password on email and bank
