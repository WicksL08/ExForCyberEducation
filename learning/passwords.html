// why password security matters

How Are Passwords Really Stored?

When you create a password on a website, it is never stored as plain text. Instead, it goes through a process called hashing — transforming it into a fixed-length string of characters that cannot be reversed. But not all password storage is secure, and attackers have developed powerful techniques to crack weak passwords.

  • Passwords stored as plain text are extremely dangerous — one breach exposes everything
  • Hashing converts passwords into fixed-length strings that cannot be reversed
  • Salting adds a random string to each password before hashing to prevent table attacks
  • Rainbow tables are precomputed hash lookups used to crack unsalted passwords instantly
  • Password reuse across sites is one of the most dangerous habits in Cyber Security

// key vocabulary

Key Terms to Know

Salting

Adding a random string to a password before hashing to prevent rainbow table attacks.

Ex: password + random_salt → hash
Rainbow Table

A precomputed table of hash values used to reverse-lookup and crack unsalted passwords.

Ex: hash lookup attack on MD5
Brute Force

Trying every possible combination of characters until the correct password is found.

Ex: trying aaa, aab, aac…
Credential Reuse

Using the same password across multiple sites — makes users vulnerable if one site is breached.

Ex: same password on email and bank
Dictionary Attack

Using a list of common words and passwords to guess a target's credentials.

Ex: trying "password123", "qwerty"
Password Entropy

A measure of how unpredictable a password is — higher entropy means harder to crack.

Ex: longer + random = higher entropy
